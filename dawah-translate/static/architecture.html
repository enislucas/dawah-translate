Component Details

Download

pipeline/download.py

Downloads YouTube video and extracts audio for Whisper.

yt-dlp FFmpeg
  • Max resolution: 1080p MP4
  • Audio: 16kHz mono WAV (PCM 16-bit)
  • Extracts metadata: title, uploader, duration
YouTube URL video.mp4 + audio.wav

Transcribe

pipeline/transcribe.py

Speech-to-text using faster-whisper on CPU.

faster-whisper CTranslate2
  • Models: large-v3 (3 GB) or medium (1.5 GB)
  • Quantization: int8 (CPU-friendly)
  • VAD filtering to skip silence
  • Auto language detection
audio.wav transcript_original.srt

Translate (Claude API)

pipeline/translate.py

Context-aware translation using Claude with glossary enforcement.

Anthropic SDK
  • Batch size: 50 segments per API call
  • Context: 5 previous segments per batch
  • Tracks introduced glossary terms
  • Character limits per segment duration
transcript_original.srt transcript_romanian.srt

Translate (NLLB Local)

pipeline/translate_local.py

Free local translation using Meta's NLLB-200 model.

CTranslate2 HuggingFace
  • Model: nllb-200-distilled-600M (~400 MB)
  • Supports: Arabic, English, Turkish, French
  • Glossary post-processing for Islamic terms
  • Known-pattern replacement table
transcript_original.srt transcript_romanian.srt

Subtitle Utilities

pipeline/subtitle.py

SRT parsing, writing, normalization, and ASS conversion.

  • Romanian diacritic normalization (ş→ș, ţ→ț)
  • Reading speed calculation (chars/sec)
  • Resegmentation: split long blocks
  • SRT → ASS with font styling

Burn Subtitles

pipeline/burn.py

Burns styled subtitles into the video using FFmpeg.

FFmpeg
  • Codec: libx264, preset medium, CRF 20
  • Font: Noto Sans Bold 56pt
  • Style: white text, black outline + shadow
  • Audio: copied (not re-encoded)
video.mp4 + romanian.srt final.mp4

Glossary System

The glossary is the key innovation for handling Islamic terminology in translation. Here's how it works in each translation path:

Claude API Path

The glossary is embedded directly in Claude's system prompt. Claude reads the rules and applies them contextually:

  • First occurrence of "صلاة" → "Salah (rugaciunea rituala)"
  • Subsequent occurrences → "Salah"
  • Tracks introduced terms across batches
  • Respects character limits

NLLB Local Path

NLLB can't be prompted, so glossary is applied as post-processing:

  • NLLB translates "صلاة" → "rugaciune"
  • Post-processor detects "rugaciune" in output
  • Replaces with "Salah (rugaciunea rituala)"
  • Uses a known-pattern table for each Arabic term
